Información

  • El escenario simula una infraestructura de red corporativa, con varias redes privadas, servidores y equipos de usuario, que ofrecen diferentes servicios vulnerables, como servicio dns, wordpress, ftp, webs con bases de datos SQL, para realizar pruebas de pentesting.
  • El punto de partida del atacante será la red interna de pruebas, necesitará comprometer los servidores y equipos de usuario o clientes conectados a dicha red, para conseguir acceder al resto de redes de la infraestructura y capturar las banderas escondidas.
  • El escenario está diseñado para lograr un aislamiento entre las diferentes redes de la infraestructura, proporcionando al usuario la posibilidad de practicar las tácticas de pivotaje entre redes y movimientos laterales entre máquinas del entorno. La máquina atacante solo tiene visibilidad de la red privada de pruebas, para pivotar a la red de preproducción será necesario utilizar la máquina vm5 como esclavo y realizar los ataques a la red de preproducción desde dicha máquina, una vez que hayamos conseguido vulnerarla. Con la red de producción se da una situación similar, en este caso se utilizarán las máquinas vm5 y vm9 como máquinas esclavo para conseguir acceso a la red.
  • El contexto que se da ha dado a este escenario reside en una máquina atacante vm1, que actúa como “insider” o amenaza interna, como podría ser un empleado deshonesto que quiere vulnerar la red corporativa y tomar el control de todas las redes y máquinas de la infraestructura.
  • El punto de partida y la entrada de la máquina virtual uno (atacante KaliLinux) al escenario está en: 192.168.56.48/29. Este rango de IPs se corresponde con el rango de la red privada de pruebas, la dificultad fácil del escenario.

vm2 Servidor web de la red de pruebas, dispone de un servicio web que muestra una página web vulnerable, llamada DVWA, que nos permitirá realizar pruebas de hacking web.
vm3 Servidor web de la red de pruebas, dispone de un servicio web que muestra una página web vulnerable, llamada Juice Shop, que nos permitirá realizar pruebas de hacking web.

vm5 Máquina de usuario de la red de pruebas y de la red de preproducción. Esta máquina nos va a servir para pivotar entre dichas redes
vm6 Servidor FTP de la red de preproducción, se encarga de la transferencia de ficheros entre los usuarios del escenario.
vm7 Servidor Windows 2008 de la red de preproducción, presenta múltiples vulnerabilidades, entre ellas el exploit eternalblue.
vm8 Máquina de usuario de la red de preproducción, presenta una vulnerabilidad en el servicio unreal ircd.
vm9 Máquina de usuario de la red de preproducción y de la red de producción. Esta máquina nos va a servir para pivotar entre dichas redes.
vm10 Servidor dns de la red de producción, se encargará de transformar las IPs de las máquinas del escenario en nombres de dominio.
